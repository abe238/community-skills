# Field Cartography

Build a map of the field, not a book report.

## Required structure
- core mental models
- canonical examples
- edge cases or boundary failures
- relation between subfields
- main tradeoffs
- recurrent error patterns

## Priorities
Favor:
- causal mechanisms
- governing abstractions
- why-models
- failure conditions

Avoid:
- chapter sequence
- name-dropping without explanatory value
- lists of terms with no operational meaning
