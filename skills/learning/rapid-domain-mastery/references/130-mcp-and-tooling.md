# MCP and Tooling Boundaries

Use MCP or host tools when you need:
- live documentation
- browser interaction
- papers or repositories not already in the corpus
- databases or APIs
- issue trackers / external systems

## Separation of concerns
- Skill = reusable workflow and expertise
- MCP = tool and data access
- AGENTS / rules = always-on persistent guidance

Do not bury tool credentials or dynamic configuration inside the skill instructions.
Keep those in the host/client configuration layer.
