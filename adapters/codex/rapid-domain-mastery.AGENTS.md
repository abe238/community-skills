# AGENTS.md — rapid-domain-mastery

Use this instruction block for Codex/OpenAI-style coding agents and any agent that reads `AGENTS.md`.

## Trigger

When the user needs to understand a new field fast from multiple sources — exam prep, research onboarding, program handover, or rapid topic mastery — follow this rule.

## Rule

Build a field map, not a summary: the expert mental model, the live debates, the hidden assumptions, the prerequisites, and the questions that separate real from fake understanding. Be honest about corpus limits — label `[Gap]` rather than papering over them.

Skip for plain summaries, single short documents, or quick factual lookups.

## Process

1. Boundaries: decide whether the corpus is strong enough to synthesize; if thin, bound claims and mark `[Gap]`.
2. Input contract: normalize subject, goal, level, time budget, source shape, and mode.
3. Corpus audit, then field cartography + mental-model extraction.
4. Debate cartography when sources disagree.
5. Assumption mining on every deep run.
6. Prerequisite ladder when foundations are missing.
7. Diagnostic questions for exam/interview/mastery prep.
8. Tutor loop or dated study sprint on request or when a time budget is given.
9. Quality gates before finalizing; watch summary drift, fake consensus, thin-corpus overreach.

## Epistemic labels

`[Consensus]`, `[Debate]`, `[Open]`, `[Prereq]`, `[Inference]`, `[Gap]`.

## Output

Corpus audit, core mental models, major disagreements, open questions, hidden assumptions, prerequisite ladder, diagnostic questions, and a tutor start or sprint plan.
